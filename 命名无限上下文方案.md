# 名义上的无限上下文：外部存储 + 检索方案

## 📊 核心原理

### 问题本质
- **大模型限制**: 单次会话 ~200K tokens
- **实际需求**: 项目级长期上下文（可能几百万 tokens）

### 解决方案
```
外部存储（文件/数据库） + 智能检索 + 摘要压缩
         ↓
   跳过模型限制
         ↓
  "名义上的无限上下文"
```

---

## 🎯 现有技能组合

### 技能地图

| 技能 | 功能 | 在无限上下文中的作用 |
|------|------|---------------------|
| `google_search` | 网络搜索 | 🔍 **检索** - 查找相关文档/代码 |
| `synth_textsummarize` | 文本摘要 | 📝 **压缩** - 生成摘要降低 token |
| `synth_jsonparse` | JSON 解析 | 📊 **结构化** - 提取关键信息 |
| `yaml_parser` | YAML 解析 | 📋 **解析** - 读取配置/元数据 |
| `autonomous_orchestrator` | 任务编排 | 🎯 **协调** - 管理多步骤检索流程 |

---

## 🚀 实现方案

### 方案 1: 文件系统驱动（推荐）

#### 架构
```
项目根目录/
├── CONTEXT.md          # 当前上下文（动态更新）
├── CONTEXT.summarized.md  # 摘要版本（压缩）
├── HISTORY.md          # 历史记录
├── HISTORY.summarized.md  # 历史摘要
├── TODO.md             # 待办事项
├── DECISIONS.md        # 决策记录
└── MEMORY/             # 内存目录
    ├── code_snippets/  # 代码片段
    ├── docs/           # 文档
    └── embeddings/     # 向量嵌入（可选）
```

#### 工作流
```
1. 用户提问 → 读取 CONTEXT.md 获取当前状态
2. 检索相关文件 → google_search / 文件扫描
3. 摘要压缩 → synth_textsummarize
4. 更新 CONTEXT.md → 保存新状态
5. 返回答案 → 基于压缩后的上下文
```

---

### 方案 2: 向量数据库 + 检索（高级）

#### 架构
```
文件 → 嵌入（Embedding）→ 向量数据库
                            ↓
                      检索相似内容
                            ↓
                      返回相关片段
```

#### 实现方式
```
1. 文件分块 → 每块 < 8K tokens
2. 生成嵌入 → 使用文本嵌入 API
3. 存储向量 → Pinecone / Weaviate / Milvus
4. 检索 → 相似度搜索 Top-K
5. 摘要生成 → synth_textsummarize
```

## 🎯 推荐实施路径

### 立即执行（Phase 1）

#### 1. 创建文件结构
```bash
# 在项目根目录执行
mkdir -p MEMORY/code_snippets MEMORY/docs MEMORY/embeddings

# 创建模板文件
touch CONTEXT.md CONTEXT.summarized.md HISTORY.md HISTORY.summarized.md
```

#### 2. 创建上下文管理技能
**目标**: 自动维护 CONTEXT.md

**能力**:
- 读取当前文件状态
- 生成摘要
- 更新 CONTEXT.md

**伪代码**:
```rust
// context_manager.rs

fn update_context() {
    // 1. 读取项目文件
    let files = scan_project_files();
    
    // 2. 生成摘要
    let summary = files
        .iter()
        .map(|f| summarize_file(f))
        .collect::<Vec<_>>();
    
    // 3. 更新 CONTEXT.md
    write_context_md(&summary);
}
```

#### 3. 配置 GEMINI.md
```markdown
# 项目层 GEMINI.md

## 上下文管理策略
1. 项目文件 → CONTEXT.md（每次会话更新）
2. 历史变更 → HISTORY.md
3. 摘要压缩 → synth_textsummarize

## 调用方式
- 更新上下文: `context_manager --update`
- 读取上下文: `cat CONTEXT.md`
- 摘要压缩: `synth_textsummarize --input CONTEXT.md`
```

### "无限上下文" = 外部存储 + 智能检索

| 组件 | 作用 | 已有技能 |

```

3. synth_textsummarize（压缩摘要）
4. 更新 CONTEXT.md（保存状态）
5. 返回答案（基于压缩上下文）
```
