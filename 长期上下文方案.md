# 长期上下文与跨项目记忆方案

## 📊 当前限制

### Gemini CLI 的上下文窗口
- **理论**: 无硬性限制（受模型 token 限制）
- **实际**: 单次会话 token 限制（约 200K tokens）
- **问题**: 跨会话无自动记忆

---

## ✅ 现有机制

### 1. GEMINI.md 模块化记忆

Gemini CLI 提供**三层 GEMINI.md**：

| 层级 | 路径 | 作用 |
|------|------|------|
| **全局层** | `~/.gemini/GEMINI.md` | 所有项目共享 |
| **项目层** | `当前项目/GEMINI.md` | 项目特定 |
| **子目录层** | `子目录/GEMINI.md` | 子目录特定 |

### 2. 自动会话保存
- **位置**: `~/.gemini/tmp/<project_hash>/chats/`
- **恢复**: `/resume` 命令
- **限制**: 仅当前会话历史

### 3. save_memory 工具
- **存储**: `~/.gemini/GEMINI.md` 的 `## Gemini Added Memories` 部分
- **范围**: 全局（所有项目）
- **示例**: 用户偏好、项目架构决策

---

## 🎯 问题分析

### 您的需求

| 需求 | 当前支持 | 说明 |
|------|---------|------|
| **项目内历史记忆** | ✅ 部分支持 | `/resume` 可恢复会话 |
| **跨项目上下文** | ❌ 无自动机制 | 需手动管理 |
| **长期状态保持** | ⚠️ 有限支持 | 依赖 GEMINI.md |
| **上下文压缩** | ✅ 支持 | `/compress` 命令 |

---

## 💡 解决方案

### 方案 1: 项目层 GEMINI.md（推荐）

**适用场景**: 单个项目长期开发

**步骤**:
```markdown
1. 在项目根目录创建 GEMINI.md
2. 添加项目上下文：
   - 项目架构
   - 技术栈
   - 开发规范
   - 已实现功能
3. 添加历史记录（增量更新）
```

**示例**:
```markdown
# Skill Router 项目上下文

## 项目状态
- **版本**: v0.2.0
- **语言**: Rust
- **最近更新**: 2026-03-13

## 已实现功能
1. ✅ 技能路由系统
2. ✅ 四阶段匹配 pipeline
3. ✅ 7 个 Rust 技能

## 当前任务
- 正在开发：...
- 待办事项：...

## 技术规范
- 使用 Rust 1.70+
- 遵循 Cargo 工作流
- ...
```

### 方案 2: 全局 GEMINI.md 维护

```markdown
- 所有项目使用统一的代码规范：...
- 项目模板路径：...
```

---

### 方案 3: 会话管理命令

**常用命令**:
```bash
# 保存会话
/chat save "任务名称"

# 列出会话
/chat list

# 恢复会话
/chat resume "任务名称"

# 压缩上下文
/compress

# 查看当前记忆
/memory show
```

---

### 方案 4: 文件驱动上下文（推荐）

**原理**: 用文件代替内存存储

**实现**:
```markdown
1. 项目根目录创建 CONTEXT.md
2. 每次会话更新：
   - 任务进度
   - 决策记录
   - 待办事项
3. 读取 CONTEXT.md 获取当前状态
```

**优势**:
- ✅ 永久保存（Git 版本控制）
- ✅ 清晰可读
- ✅ 易于共享

---

## 📁 推荐文件结构

```
项目根目录/
├── GEMINI.md          # Gemini CLI 指令
├── CONTEXT.md         # 当前上下文（动态更新）
├── TODO.md            # 待办事项
├── DECISIONS.md       # 决策记录
├── HISTORY.md         # 历史记录
├── README.md          # 项目说明
└── CHANGELOG.md       # 变更日志
```

## 🎯 总结

### 没有"无限上下文"，但有：

1. ✅ **项目层 GEMINI.md** - 项目特定上下文
2. ✅ **全局 GEMINI.md** - 跨项目偏好
3. ✅ **会话命令** - 恢复/压缩/保存
4. ✅ **文件驱动** - CONTEXT.md 模式

### 关键：建立文件驱动工作流

**让文件代替内存存储状态**：
- `CONTEXT.md` - 当前状态
- `TODO.md` - 待办事项
- `DECISIONS.md` - 决策记录
- `HISTORY.md` - 历史变更

这样**即使换会话、换电脑，上下文也不会丢失**！
