### 6.8 三引擎并发编排稳定性修复

本节记录对“三引擎并发编排不稳定超时”的进一步修复与真实复验。

#### 6.8.1 根因定位

进一步排查后确认，问题不是单一来源，而是两层叠加：

1. **参数字段不一致**
   - `ai_parallel_solve` / `ai_triple_vote` 的源码优先读取 `problem`
   - 但外部很多调用会传 `task`
   - 当只传 `task` 时，builtin 可能退回到 `ctx.task`，导致实际 prompt 变成类似：
     - `ai_parallel_solve: {"engines":[...],"task":"..."}`

2. **Claude CLI 会话污染**
   - 在未显式关闭会话持久化时，Claude 可能带入旧上下文或旧权限语境
   - 真实现象是：同一个极简任务里，Claude 返回了与当前问题无关的权限提示，而不是超时或目标答案

#### 6.8.2 源码修复

修复内容：

1. `ai_parallel_solve` 同时支持 `problem` 和 `task`
2. `ai_triple_vote` 同时支持 `problem` 和 `task`
3. Claude 子进程调用追加稳定参数：
   - `--model sonnet`
   - `--no-session-persistence`
   - `--disable-slash-commands`
