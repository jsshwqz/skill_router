# 🟢 Aion-Forge 技能使用手册 (全家桶指南)

> **注**: 这份文档是专门为您（人类）和您的 AI 助手们准备的。它解释了每个“技能”能干什么，以及怎么用。

---

## 🧭 核心概念：什么是“技能” (Capability)?
在 Aion-Forge 中，**技能**是系统能做的最小功能单元。你可以把它想象成一个乐高积木，AI 会根据你的指令，自动把这些积木拼起来完成任务。

---

## 🛠️ 第一类：内容解析专家 (能读懂各种文件)

我们将复杂的格式转化成 AI 容易处理的数据。

### 1. YAML 解析 (`yaml_parse`)
*   **白话解释**: 帮 AI 读懂那些长得像大纲一样的配置文件。
*   **适用场景**: 当你有 `.yaml` 或 `.yml` 格式的任务列表、集群配置时。
*   **输入示例**:
    ```yaml
    任务: 喝水
    次数: 3
    ```
*   **输出结果**: 转换成结构化的数据，方便后续计算。

### 2. JSON/TOML/CSV 解析
*   **JSON**: 互联网通用语言，非常精确。
*   **TOML**: 看起来很整洁的配置文件。
*   **CSV**: 电子表格（Excel）的纯文本版本。

---

## 🧠 第二类：灵动脑力工具 (自然语言处理)

让系统具备像人一样的理解和翻译能力。

### 1. 文本摘要 (`text_summarize`)
*   **白话解释**: “太长不看”过滤器。把万字长文变成三句话。
*   **输入**: 一篇长文章。
*   **输出**: 核心要点总结。

### 2. 差分对比 (`text_diff`)
*   **白话解释**: 找不同。
*   **输入**: 代码的版本 A 和版本 B。
*   **输出**: 哪一行删了，哪一行加了（变动一目了然）。

---

## 🛠️ 第三类：开发者之手 (代码辅助)

专门为 Rust 开发量身定制。

| 技能名称 | 它能帮你做什么？ | 产出物 |
| :--- | :--- | :--- |
| **代码生成** | 你说需求，它写 Rust 代码 | `.rs` 源文件 |
| **单元测试** | 自动给你的代码写测试用例 | `#[test]` 代码块 |
| **代码评审** | 像导师一样检查你的 Bug | 改进建议报告 |

---

## 🔍 第四类：奇迹搜索与进化 (核心黑科技)

这是本系统的“灵魂”，让它可以学习新技能。

### 1. 级联搜索 (`discovery_search`)
*   **白话解释**: 当系统遇到“不会干”的任务时，它会偷偷去 Google、找本地资料、翻网页，把别人怎么做的抄回来。
*   **价值**: 它是系统**自我进化**的源泉。

### 2. 长期记忆 (`memory_remember/recall`)
*   **白话解释**: 记事本。系统会记住你以前报错的地方、你的偏好。
*   **效果**: 下次再做类似任务，它会变得比上次更聪明。

---

## 💡 给 AI 助手的悄悄话 (AI Interaction Tips)
如果你是另一个 AI，想调用这些技能，请遵循以下原则：
1. **看 Schema**: 所有的输入参数必须符合 `manifest.json` 里的 JSON 格式。
2. **看 示例**: 在 `examples` 字段里有标准的 Payload 模板，直接复制改改就能用。
3. **验证**: 如果技能执行报错，请调用 `code_test` 技能来生成对应的修复脚本。

---

### 如何快速查看最新列表？
在终端运行：
```bash
cargo run -p aion-cli -- export-manifest
```
它会生成一个 `manifest.json`，那是给机器看的“说明书”。
